Docker学习总结

发表于 2017-05-07

学习Docker的小小总结

　　这篇文章主要是对Docker概念的总结，对具体使用没什么介绍，希望大家有针对性的看，希望看到介绍如何使用的同学请绕行。

虚拟机和容器

　　很多同学大概对Docker还没有一个清晰的了解，其实Docker就是一种Linux容器，它独特的架构方式和工作流程使得Linux容器走入普通技术人员的生活中。或许有些同学对Linux容易也不是很熟悉，没关系，你看到文章标题能点进来的话，我猜至少你对虚拟机的概念应该还算熟悉，生活中我们会在windows系统里用虚拟机跑个ubuntu的系统，在mac笔记本上装个Windows这样的，你们可能用到过，对吧。
　　常用的虚拟化系统有VMware或Xen，它们的作用是在虚拟层之上运行完整的内核和操作系统，使用这种方式运行的虚拟机之间隔离的很好，各个虚拟机的内核运行在不同的内存>空间中，而且各自定义了进入物理硬件的入口点。我在自己的mac上，可以装个VMware，里面可以跑一个Ubuntu，再跑一个Windows，他们可以同时运行，互不干扰。
　　容器则是完全不同的虚拟化方式，所有容器共用一个内核，而且容器之间的隔离完全在这个内核中实现这种操作叫做操作系统的虚拟化。容器的最大优势是高效使用资源，因为不用为了使用各个独立的功能而运行整个操作系统。但由于共用内核，隔离执行的任务和底层硬件之间少了一层交互，只能运行与底层内核兼容的进程，因此Windows应用是不可以>在Linux容器中运行的。所以，最好把容器视为一项Linux领域的技术，看作为服务器中进程的一种包装，丢掉以往对于虚拟机的认识。
　　讲到这里，大家应该对容器有一个很直观的认识，它就是一种“轻量级”的虚拟机，只虚拟化操作系统，不虚拟内核，可能我这个比喻不是很恰当，但我觉得对于刚开始接触的人来说有助于理解，在一个Linux系统中，我们可以运行多个Linux容器，在多个容器中分别提供多个相对独立的服务。正是由于这种”轻量级“，容易占用的空间非常少，且启动要快得多。而今天的主要要介绍的Docker，就是将Linux容器与其他一些技术组合而成的一种工具。

Docker的作用

　　讲了容器和虚拟机的区别，大概知道了Docker是个什么东西，你大概想知道Docker到底是用来干什么的，有什么效果。我大概总结了这么几个：

简化业务流程

　　在以往部署应用的过程中，需要开发团队和运维团队的沟通与配合，这个过程是个效率低，费时的过程，这个过程还会降低开发团队日后的创新能力，因为如果部署软件很难需要其他部门提供资源，开发者通常会把所有东西塞到现有的应用中，以免下次部署再带来痛苦。 Docker可以使用映像仓库作为中转站，把构建应用映像的职责与部署和操作容器的职责区分开。在实践中，这意味着开发团队可以连同应用所有依赖一起构建应用，在开发环境和测试环境中运行之后，再把完全一样的应用和依赖构建包，上传到生产环境。从外部看，构建包都是一样的，运维团队可以构建或安装标准的工具，用于部署运行应用。

一次编写，到处运行

　　由于支持Docker的云平台越来越多，Docker正在变成云服务提供商的通用格式。2014年一个Orchard公司的工程师做了演示，将一个Docker容器部署到多个不同云平台中，而之前这样的业务流程无法轻易实现。

改变使用应用的方式

　　Docker容器使以前使用的一体式应用栈可以分解为几十个甚至上百个作用单一的小型应用，这些小型应用的组合功能和一体式应用完全相同，然而这些小应用更易于重写，更易于重用和管理。

Docker适合场景和注意事项

　　上一节讲了Docker的用途，但是与大多数工具一样，它也有自己适合的场景，那就是无状态的应用。适合使用Docker的应用包括Web前端，后端API和短期运行的任务，如cron脚本。有的应用也是需要存储状态的，我们会将状态外存，如果是配置数据，最好通过环境变量传入，Docker原生支持环境变量，环境变量存储在组成容器配置的元数据中，因此，每次重启容器都会把相同的配置传给应用。可弹性伸缩的应用通常在数据库中存储状态，Docker并不阻止容器花的应用这么做，可是如果应用存储文件的话就有难度，在容器的文件系统中存储数据性能不好，会受到空间限制，而且在容器生命周期内无法保有状态，这种情况要慎重。虽说可以通过外存来保存状态，但Docker社区强烈建议从不需要持久化状态的应用入手。Docker中文社区

总结

　　这篇总结来自于看了Docker: Up and Running的一些笔记。总的来说，Docker就是一种Linux容器，只是加入了一些>其他技术使得容器更容易被使用，它带动了一股浪潮。Docker适合无状态应用的部署，简化了部署流程，便于应用的细分。引用Lars Herrmann的话叫：新瓶装旧酒，风味尤佳。
